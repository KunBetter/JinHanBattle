# 汉金之战 — UI/视觉/战斗体验升级设计文档

**日期**: 2026-05-30  
**状态**: 设计确认，待实施

---

## 一、概述

本轮升级聚焦四个维度：UI 界面重设计、战场氛围、单位表现力、战斗反馈。设计方向为 **古卷画轴风**——以卷轴/竹简/兵符/奏章等古代军事文书为 UI 载体，最大化古风沉浸感。

---

## 二、UI 界面重设计（第一优先级）

### 2.1 顶栏 → 指挥卷轴

**概念**：横向展开的古卷轴，两端木质轴杆，卷面宣纸纹理，信息按层级排列。

**布局**（左→右）：
- 左侧：标题"漢金之戰"+ 装饰分隔线
- 中央：阵营 VS 徽章（漢⚔金，红蓝双色）
- 右侧：天气（图标+名称）、连击（数字+金色）、时辰（计时器）、金币（金框独立容器）

**视觉特征**：
- 木质轴杆：CSS 渐变模拟（#6B3A2A → #A0704A → #8B5A3A）
- 卷面：半透明暖色底 `rgba(245,235,200,0.12-0.18)`
- 信息层级：小标签 10px 灰色 + 大数值 14-16px 金色/白色，上下排列
- 金币：独立金框 `border:1px solid rgba(240,214,138,0.25)` + `rgba(240,214,138,0.1)` 背景

**涉及文件**：`index.html`（HTML 结构调整）、`style.css`（新增轴杆/卷面/徽章样式）

### 2.2 技能栏 → 竖排兵符

**概念**：4 个技能按钮改为竖向排列的兵符令牌，从左到右按威力递进。

**布局**：
- 火攻（红底）、夜战（蓝底）、传令（绿底）— 72px 宽
- 诏令（金底）— 88px 宽，大一号，`box-shadow: 0 0 20px rgba(212,168,75,0.2)`

**每个兵符包含**：
- 技能图标（24-28px emoji）
- 技能名称（13-15px）
- 金币费用（◆ 标记 + 数字）
- CD 进度条（3px 高，底色 + 渐变色填充）
- CD 状态文字（"就绪"绿色 / "CD 21s"橙色 / "CD 48s"红色）

**选中/可用状态**：
- 可用：对应阵营/技能色调边框 + 微光
- 禁用：`opacity:0.35` + `filter:grayscale(0.5)`（保留现有逻辑）

**涉及文件**：`index.html`（skill-bar 结构调整）、`style.css`（兵符卡片样式）、`game.js`（CD 进度条宽度动态更新）

### 2.3 部署栏 → 令牌阵

**概念**：9 个兵种卡片排列，每张是一块"兵符令牌"，2 行布局。

**每张令牌**：
- 兵种图标（20px emoji）
- 兵种名称（11px）
- 费用（◆ 标记 + 数字，金色）

**选中状态**：
- `translateY(-3px)` 翻起效果
- 金色边框 + `box-shadow: 0 0 16px rgba(212,168,75,0.3)`
- 底部"▲ 选中"小字提示

**点数显示**：
- 独立容器："兵符"标签 + 大号金色数字 + "/ 60"
- 与自动布阵/出战按钮垂直排列在令牌阵右侧

**按钮**：
- "自动布阵" — 辅助按钮样式（灰色）
- "出 战" — 红色渐变主要按钮

**涉及文件**：`index.html`（troop-bar 结构调整）、`style.css`（令牌卡片样式）

### 2.4 胜利画面 → 战报奏章

**概念**：宣纸质感的奏章从画面中央展开，加盖将军印，逐行浮现战果。

**奏章视觉**：
- 宣纸渐变背景 `#F5ECD7 → #E0D4BC → #C8B898`
- 三层金色边框模拟奏章封套 `box-shadow: 0 0 0 8px #8B6914, 0 0 0 10px #A0782C, 0 0 0 13px #6B4E0A`
- 顶部/底部装饰线 `border-top/bottom:1px solid #8B6914`
- 标题"大 捷"（36px 深棕色）
- 红色圆形"勝"字印章（`rotate(-15deg)`）

**统计面板**：
- 用时 / 歼敌 / 城门 / 最高连击 — 左标签右数值
- 赏金单独一行，金色大号字体
- 虚线分隔

**成就徽章**：
- 金色圆角标签 `background:#F0D68A; color:#5D2E0C`
- 水平排列在统计下方

**动画序列**：
1. 展开（0.6s）：奏章从中央水平线展开，背景变暗
2. 落印（0.4s）：印章旋转+缩放弹入
3. 逐行浮现：统计数据每行 0.15s 间隔出现

**两种结局**：
- 汉军大捷（攻方胜）：红色"勝"印 + 暗金城墙崩塌粒子
- 金军坚守（守方胜）：蓝色"固"印 + 铁壁闪光粒子

**涉及文件**：`index.html`（overlay-victory 重写）、`style.css`（奏章样式+动画）、`game.js`（动画序列控制）

## 三、战场氛围提升（第二优先级）

### 3.1 天气可视化强化

**每种天气的具体强化**：

| 天气 | 强化内容 |
|------|----------|
| ☀️ 晴 | 热浪波纹（canvas 正弦扭曲叠加 2-3 条）、阳光光柱（径向渐变半透明） |
| 🌧️ 雨 | 雨滴溅射（击中单位时 2-3 个蓝色微粒）、地面积水反光（底部 280-550 加 0.05 蓝色叠加）、云层加厚（天空渐变变暗 20%） |
| 💨 风 | 沙尘横向飘过（dust 粒子，y:280-500）、旗帜摆动角度增大（从 ±5° 到 ±25°）、远程弹道轨迹弯曲（catapult/spear 攻击线画弧线） |
| ⛈️ 雷 | 闪电划破天空（白色锯齿折线 + 全屏闪白 0.1s）、雷击点地面焦痕（3 秒黑色圆斑淡出）、暴雨密度加倍（rain 粒子 ×2） |

**实现方式**：在 `_drawSky()` 和 `_updateBattle()` 中按天气类型增加分支逻辑，不创建新类。

**涉及文件**：`game.js`（_drawSky、_updateBattle 中天气分支扩展）

### 3.2 战场氛围层

**城堡烽火**：
- 城墙上 2-3 处火盆（x:840/880/920, y:150）
- 火焰粒子持续生成（fire 类型，每 0.25s 2-3 个）
- 黑烟粒子持续升腾（smoke 类型，颜色加深）

**行军尘土**：
- 移动中的单位在其身后生成 dust trail 粒子
- 触发条件：单位 speed > 0.5 且处于移动状态
- 每 0.3s 在单位位置生成 1 个棕色 dust 粒子

**击杀痕迹**：
- 单位死亡点留下地面焦痕/血迹
- Canvas 绘制 3-5 秒淡出的暗色椭圆 + 暗红渐变
- 存储在 `this._killMarks` 数组中，类似 damageNumbers 的生命周期管理

**飞鸟惊起**：
- 诏令释放时，城堡后方飞出 5-8 个深色三角形
- 向右上方抛物线运动，1.5 秒后消失
- 纯装饰，不影响游戏逻辑

**涉及文件**：`game.js`（新增 _drawBattleAtmosphere 方法，在 _render 中 _drawTerrain 之后调用）

## 四、单位表现力提升（第三优先级）

### 4.1 兵种剪影差异化

**骑兵**：
- 在现有 48×56 精灵图中增加马匹轮廓（下半部分马蹄 + 马身）
- 马身颜色与阵营色协调（汉=深棕马，金=灰白马）

**冲车**：
- 增加木制撞锤结构：水平横梁 + 前方金属撞头（圆形带尖刺）
- 多人推车剪影：2-3 个小人形在撞锤后方

**投石车**：
- 增加大型木架：A 字形支架 + 横梁 + 投臂（可动部件）
- 石弹绘制：圆形石头放在投臂末端

**盾兵**：
- 塔盾面积扩大（覆盖从肩到膝）
- 移动时盾牌微晃（在精灵图内偏移 1-2px）

**军师**：
- 增加羽扇（手持）+ 长袍（下半身梯形扩展）
- 长袍底部飘逸效果（bob 动画幅度加大）

**实现方式**：修改 `assets.js` 中 `_drawArmsAndWeapon`、`_drawBody`、`_drawLegs` 中对应兵种的绘制分支。

**涉及文件**：`assets.js`（5 个兵种的 detail 绘制函数扩展）

### 4.2 单位头顶 UI 增强

**状态图标**：
- 增益 buff（atkBuff/spdBuff）：血条上方显示小三角形 ▲ 和颜色标记（atk=红, spd=蓝）
- debuff（enemySlow/夜战减速）：血条上方显示 ▼ 标记
- 绘制位置：血条 `barY - 6` 处

**血条分段式**：
- 10 段小格代替连续长条，每段 3.4px 宽 + 0.5px 间距
- 损失的血格：从右向左变成暗色（不是瞬间消失，而是闪烁 0.3s 后暗掉）

**仇恨连线**：
- 选中的己方单位与攻击目标之间画淡色虚线
- `ctx.setLineDash([2, 4])` + `strokeStyle:rgba(255,255,255,0.15)`
- 仅当选中的单位正在攻击时显示

**军师光环可视化**：
- 军师周围画虚线圆（auraRange=80px 半径）
- `strokeStyle:rgba(255,255,255,0.12)` + `setLineDash([3, 6])`
- 光环范围内友军有微弱同色轮廓光（1px 外发光）

**涉及文件**：`game.js`（_drawUnits 方法扩展，约 100-150 行新增）

## 五、战斗反馈强化（第四优先级）

### 5.1 伤害数字风格化

在现有 `_addDamageNum(x, y, value, isCrit)` 基础上增加 type 参数：

| 类型 | 视觉 | 行为 |
|------|------|------|
| 普通 | 白色 13px | 上飘 60px，1s 淡出 |
| 暴击 | 金色 18px + "!" 后缀 + 描边 | 上飘 80px，先放大再缩小，1.2s 淡出 |
| 格挡 | 灰色 11px + "🛡" 前缀 | 上飘 40px，0.8s 淡出 |
| 灼烧 | 橙色 12px + "🔥" 前缀 | 波浪式上飘（sin 水平偏移），1s 淡出 |
| 治疗 | 绿色 13px + "+" 前缀 | 缓飘上浮 40px，1.5s 淡出 |
| 击杀 | 红色 20px "击杀" 标签 | 屏幕微震（shake+2），弹入+淡出 1.5s |

**涉及文件**：`game.js`（_addDamageNum 签名扩展 + _drawDamageNumbers 样式分支）

### 5.2 连击系统升级

**连击等级文字**：
- x3 → "势如破竹"（白色）
- x5 → "所向披靡"（金色）
- x8 → "万夫莫敌"（橙色+描边）
- x12 → "天下无双"（红色+发光+震动）

**连击音阶**：
- SoundManager 新增 `comboUp(level)` 方法
- 每个连击等级对应一个音高：C→E→G→C（do-mi-sol-do 上行音阶）
- 断连时播放短促下降音

**连击进度环**：
- combo 文字周围 Canvas 绘制环形倒计时
- 5 秒倒计时，环形按比例缩小（从 360° 到 0°）
- 剩余 < 1 秒时环形变红

**涉及文件**：`game.js`（combo 显示逻辑 + 环形绘制 + SoundManager 扩展）

## 七、技术约束

- 不引入任何第三方框架或库
- 所有渲染保持在 Canvas 2D + CSS 范围内
- 不增加外部资源文件（图片/音频）
- 保持 1100×550 画布尺寸不变
- 兼容现有 localStorage 存档格式
- 程序化音效使用 Web Audio API（已有 SoundManager 类）

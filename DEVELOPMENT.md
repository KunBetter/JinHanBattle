# 汉金之战 — 游戏开发全案

## 一、创意概念

### 1.1 概述

**汉金之战**是一款基于 HTML5 Canvas 的古代峡谷攻城网页游戏。玩家扮演汉朝（攻方）或金国（守方）的将军，在峡谷战场上调兵遣将，通过部署不同兵种、施放技能、应对天气变化，最终摧毁城门（攻方）或歼灭敌军（守方）来取得胜利。

### 1.2 核心玩法

- **阵营选择**：汉（攻方）vs 金（守方），双方目标不同
- **策略部署**：有限的部署点数（15 点），合理搭配兵种阵容
- **实时战斗**：单位自动寻敌、攻击，玩家施放技能影响战局
- **动态环境**：天气随机变化，地形影响战术，宝箱带来变数

### 1.3 技术选型

| 项目 | 选型 |
|---|---|
| 游戏引擎 | 纯 JavaScript ES6，无框架 |
| 渲染 | HTML5 Canvas 2D (1100×550) |
| 音效 | Web Audio API 程序化生成 |
| 存档 | localStorage |
| 构建 | 无构建工具，单文件 script 引入 |
| 布局 | CSS Flexbox + 自定义属性 |

### 3.4 粒子系统（7 种类型）

| 类型 | 用途 | 视觉 | 物理行为 |
|---|---|---|---|
| fire | 火攻技能、击杀 | 白→橙→红径向渐变 | 上升、膨胀 |
| spark | 击杀、雷击 | 金色光点 + shadowBlur | 加速上升、水平减速 |
| rain | 雨天、雷暴 | 蓝色斜线 | 重力下坠 |
| dust | 击杀、传令抵达 | 棕色半透明圆 | 缓慢上升膨胀 |
| star | 夜战、诏令、宝箱 | 白色核心 + 金色渐变光晕 | 正弦振荡 |
| smoke | 通用 | 灰色半透明圆 | 膨胀、全方位减速 |
| confetti | 宝箱拾取 | 彩色矩形 | 重力下坠、正弦水平漂移 |

### 3.5 地形系统

| 类型 | HP | 特性 |
|---|---|---|
| 岩石 | 100 | 不可通行，可被攻击摧毁（6 处随机分布） |
| 拒马 | 60 | 不可通行，可被攻击摧毁（3 处） |
| 高地 | ∞ | 可通行，站上去 +20% 射程 +10% 攻击（3 处） |

单位移动受阻时自动攻击最近的障碍物。

### 3.6 宝箱系统

每 15~25 秒在地面随机生成一个宝箱（20 秒后消失），单位走进 32px 自动拾取。4 种奖励：

| 奖励 | 概率 | 效果 |
|---|---|---|
| 💰 金币 +20~50 | 50% | 直接增加金币 |
| ❤️ 全体回血 +10~25 | 25% | 所有己方存活单位回复 HP |
| ⚔️ 攻击力 +50% | 15% | 持续 10 秒 |
| 💨 速度 +30% | 10% | 持续 10 秒 |

### 3.7 音效系统（12 种）

全部使用 Web Audio API 程序化生成，无需外部音频文件：

- **hit**: 方波，300→80Hz 快速衰减
- **crit**: 锯齿波，600→200Hz 强力音色
- **kill**: 三角波，500→150Hz 中长衰减
- **gateHit**: 三角波低频底音 + 正弦波震动
- **combo**: 三连正弦上升音阶
- **fire**: 白噪声 0.8s
- **night**: 正弦波 200→60Hz 长滑音
- **messenger**: 三角波 C-E-G 三连音
- **decree**: 锯齿波扫频 80→400→60Hz
- **weather**: 正弦波短促上升
- **chest**: 正弦波 C-E-G-C 四连音阶上行
- **victory/defeat**: 三角波旋律上行/下行

## 五、技术要点

### 5.1 游戏循环架构

```
requestAnimationFrame
  └── 仅 BATTLE 状态执行 _updateBattle(dt)
  └── 每帧执行 _render()
       ├── 屏幕震动变换
       ├── _drawSky()      ← 支持白天/夜晚/雨天/风暴
       ├── _drawCanyon()
       ├── _drawCastle()
       ├── _drawDeployZone()
       ├── _drawTerrain()   ← 岩石/拒马/高地
       ├── _drawUnits()     ← 9 种兵种
       ├── _drawChests()    ← 宝箱脉冲光晕
       ├── _drawDeathAnimations()
       ├── _drawParticles() ← 7 种粒子类型
       ├── _drawDamageNumbers()
       └── 城门闪光效果
```

### 5.2 战斗更新循环

```
_updateBattle(dt)
  ├── 计时器、连击衰减、特效衰减
  ├── 天气更新（切换、持续特效、雷击伤害）
  ├── 宝箱生成与拾取判定
  ├── 拾取 buff 衰减
  ├── 单位更新循环
  │    ├── 军师光环检测
  │    ├── 寻敌/寻城门
  │    ├── 攻击判定（射程含地形/天气/光环修正）
  │    └── 移动（速度含天气/夜战/buff/光环修正 + 地形碰撞）
  ├── 伤害数字更新
  ├── 粒子物理更新
  ├── 技能更新（CD、持续、火攻灼烧、按钮 UI）
  └── 胜负判定
```

### 5.3 性能优化

- 粒子数组每帧 filter 清理过期粒子
- 伤害数字同粒子策略
- 死亡单位 hp <= 0 过滤，不参与战斗循环
- Canvas 仅在状态切换时重绘必要元素
- 音效延迟初始化，仅首次使用时创建 AudioContext

### 5.4 关键设计决策

1. **无框架**：纯 JavaScript ES6，零依赖，打开 index.html 即可运行
2. **单文件架构**：所有游戏逻辑集中在 game.js，降低加载复杂度
3. **emoji 渲染**：用系统 emoji 替代精灵图，跨平台一致性好
4. **程序化音效**：无需加载音频文件，体积为 0
5. **Canvas 2D**：1100×550 固定画布，响应式缩放适配屏幕
